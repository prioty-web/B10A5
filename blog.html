Important FAQ

What is the Document Object Model (DOM)?

Web documents include a programming interface called the Document Object Model (DOM). It depicts a webpage's structure as a tree of objects, with each element, attribute, and text segment acting as a node. The DOM is a tool used by developers to dynamically access, alter, and work with online information.

How do you select an element from the DOM?

There are several methods depending on our need:

document.getElementById('elementId');
document.getElementsByClassName('className');
document.getElementsByTagName('div');
document.querySelectorAll('div.className');

What is event delegation in the context of the DOM, and why is it useful?

In the DOM, event delegation is a technique where a parent element's single event listener is attached to handle events that are triggered by its child elements. Instead of adding multiple listeners to individual child elements, the parent listens for events that bubble up from its children.

Why It is Beneficial:
Efficiency: When there are numerous child elements, having a single listener instead of multiple ones uses less memory.
Managing dynamic content: Gathers events for newly added DOM elements automatically.
Code simpler: centralised event handling minimises complexity.
